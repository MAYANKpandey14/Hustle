# Code-Review Skill

1. Scope of Action: It should create detailed report and then start making surgical fixes as per the report
2. Stack Specificity: It should cover any language or framework
3. Report Structure: Markdown, detailed, code snippets where issues are found and implementation plan and testing plan.

I'm thinking:

- Executive Summary (The "Big Picture" health check)
- Architectural Risks (Scalability, Coupling, State Management)
- Performance & Efficiency (O(n) issues, N+1 queries, memory leaks)
- Production Readiness (Observability, Error handling, Security)
- Actionable Roadmap (Priority-ranked improvements)
4. Triggering: Review the Codebase.
